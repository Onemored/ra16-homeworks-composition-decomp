Декомпозиция
===

Вы работаете в стартапе, который решил тягаться с самим Яндексом в части предоставления контента. Это, конечно же, шутка, но задача нешуточная.

Вам принесли дизайн-макет, похожий на этот:

![](./assets/decomposition.png)

Что вам нужно сделать:

1. Разбейте весь интерфейс на компоненты и в файле каждого компонента напишите буквально одну строку комментария, за что данный компонент отвечает. Можете использовать формат TSDoc, детали про него можно посмотреть по [ссылке](https://tsdoc.org/). Также рекомендуем [статью](https://typescriptlang.org/docs/handbook/jsdoc-supported-types.html) на тему документирования TypeScript.
2. Постарайтесь повторяющиеся компоненты сделать настраемыми за счёт `props`. Допустим, у каждой новости в списке новостей: иконка, текст и ссылка.

Функциональность и стилизацию реализовывать не нужно, достаточно базового оформления, чтобы видно было все блоки.

---


## Реализация

Интерфейс собран на React из независимых компонентов для новостей, котировок,
промоблока, сервисов, поиска, рекламы и информационных виджетов. Повторяющиеся
элементы получают содержимое через `props`. Страница адаптируется к экранам от
320 px, поддерживает клавиатурную навигацию и настройку `prefers-reduced-motion`.

## Требования

- Node.js 24 LTS (минимально совместимая версия — 20.19);
- Yarn 1.22 или новее;
- Python 3 для создания локального виртуального окружения.

## Установка и запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
yarn install
yarn dev
```

После запуска Vite выведет локальный адрес приложения.

## Команды

```bash
yarn dev
yarn build
yarn preview
yarn lint
yarn test
yarn test:coverage
yarn validate
```

`yarn validate` последовательно запускает ESLint, тесты Vitest со 100% порогами
покрытия и production-сборку Vite.

## Публикация

В Vite используется относительный `base`, поэтому JavaScript, CSS и изображения
корректно загружаются из подпапки репозитория на GitHub Pages.
